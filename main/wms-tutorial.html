1. 시스템 소개 및 특징

메이크창고 WMS는 창고 내 팔렛트 위치를 시각적으로 관리하는 웹 기반 창고관리 시스템입니다. 별도 설치 없이 스마트폰·태블릿·PC 브라우저에서 바로 사용할 수 있으며, 모든 기기에서 데이터가 실시간으로 공유됩니다.

☁️
실시간 클라우드 동기화
Firebase 기반으로 어떤 기기에서 작업해도 즉시 반영됩니다.
📱
모바일 최적화
핀치 줌·터치 드래그로 창고 배치도를 손쉽게 조작합니다.
🗓️
유통기한 관리
만료·임박(90일 내) 팔렛트를 색상으로 즉시 식별합니다.
📊
엑셀 내보내기
재고 현황을 CSV로 다운로드하여 보고서에 활용합니다.

2. 로그인 및 최초 설정

최초 접속 시 (비밀번호 설정)

  1. 1앱에 처음 접속하면 비밀번호 설정 화면이 나타납니다.
  2. 2새 비밀번호를 입력합니다 (4자리 이상).
  3. 3비밀번호 확인란에 동일하게 재입력합니다.
  4. 4[설정하고 시작하기] 버튼을 누르면 완료됩니다.
⚠️ 중요: 비밀번호는 해당 기기에 저장됩니다. 잊어버린 경우 로그인 화면 하단의 "비밀번호를 잊으셨나요?"를 눌러 초기화할 수 있습니다. (단, 데이터는 유지됩니다)

일반 로그인

항목내용
아이디makechango (고정, 변경 불가)
비밀번호최초 설정한 비밀번호 입력
💡
여러 직원이 같은 앱을 사용할 경우 아이디와 비밀번호를 공유하여 사용합니다. 각 기기에서 최초 1회만 설정하면 이후에는 자동으로 입력됩니다.

3. 창고 만들기

  1. 1메인 화면 우측 상단 [+ 새 창고] 버튼을 누릅니다.
  2. 2창고 이름을 입력합니다. (예: 본창고 A동, 냉동창고)
  3. 3행 수와 열 수를 입력합니다. (2~30 사이, 기본값 10×10)
  4. 4[생성하기] 버튼을 누르면 격자 형태의 빈 창고가 생성됩니다.
💡 행/열 수 설정 기준: 실제 창고의 팔렛트 구역 수에 맞게 설정하세요. 예를 들어 가로 5줄, 세로 8줄 구조라면 행 8 × 열 5로 설정합니다.

4. ⚙️ 설정 탭 — 창고 구조 설계

창고를 처음 만든 후 반드시 설정 탭에서 각 칸의 유형을 지정해야 합니다.

셀(칸) 유형

유형색상용도
🔵 랙파란색팔렛트를 실제로 보관하는 구역 (가장 많이 사용)
🟡 보관노란색선반 등 소량 보관 구역
⚪ 빈공간회색통로, 작업 공간 등
🔲 미사용어두운 회색사용하지 않는 구역

설정 방법

  1. 1상단에서 원하는 유형(랙·보관·빈공간·미사용)을 선택합니다.
  2. 2분할 피커에서 한 칸을 몇 개 슬롯으로 나눌지 설정합니다. (예: 2×3 = 1칸에 6팔렛트)
  3. 3그리드에서 설정할 칸을 탭합니다. (여러 칸 동시 선택 가능)
  4. 4[적용 (N칸)] 버튼을 눌러 반영합니다.
  5. 5설정 완료 후 상단 [저장] 버튼을 눌러 저장합니다.
💡
[전체선택] 버튼으로 모든 칸을 한 번에 선택하고 일괄 적용할 수 있습니다. 전체를 랙으로 설정한 후 일부를 빈공간·미사용으로 바꾸는 방식이 빠릅니다.
분할(divR × divC) 활용 예시
• 1×1 : 칸 1개 = 팔렛트 1개
• 2×1 : 칸 1개를 세로 2분할 = 팔렛트 2개
• 2×3 : 칸 1개를 6분할 = 팔렛트 6개 (소형 창고에 유용)

5. 📦 작업 탭 — 입고 등록

입고는 ① 입고 등록 → ② 슬롯 배치 2단계로 이루어집니다.

입고 등록 방법

  1. 1창고 화면에서 [작업] 탭을 선택합니다.
  2. 2상단 툴바의 [입고] 버튼을 누릅니다.
  3. 3입고 정보를 입력합니다.
항목설명예시
상품명입고되는 상품의 이름냉동삼겹살, 참치캔
팔렛트 수입고되는 팔렛트 개수5
팔렛트당 수량팔렛트 1개의 상품 수량100 (개)
유통기한YYMMDD 6자리 형식261231 → 2026.12.31
  1. 4[입고 등록] 버튼을 누르면 하단 입고대기 목록에 추가됩니다.
💡
유통기한을 입력하면 바로 아래에 변환된 날짜가 표시됩니다. 입력 전에 확인하세요.
예: 261231 → 2026.12.31

6. 📦 작업 탭 — 입고 배치

입고 등록 후 실제 창고의 빈 슬롯에 팔렛트를 배치합니다.

방법 A: 자동 배치 (빠른 방법)

  1. 1하단 입고대기 목록에서 상품 버튼을 탭합니다.
  2. 2첫 번째 빈 슬롯에 자동으로 배치됩니다.

방법 B: 수동 위치 지정 (권장)

  1. 1그리드에서 배치할 빈 슬롯을 탭합니다. (파란 숫자 ①②③ 표시)
  2. 2여러 슬롯을 순서대로 탭하여 다중 선택합니다.
  3. 3하단에서 배치할 상품 버튼을 탭합니다.
  4. 4선택한 슬롯 순서대로 자동 배치됩니다.
✅ 다중 배치 예시: 슬롯 ①②③ 선택 후 "참치캔" 탭 → ①에 배치.
다시 "참치캔" 탭 → ②에 배치. 다시 탭 → ③에 배치. 순서대로 자동으로 채워집니다.
⚠️
배치 완료 후 반드시 [저장] 버튼을 눌러주세요. 저장하지 않으면 다른 직원의 화면에 반영되지 않습니다.

7. 📦 작업 탭 — 팔렛트 이동 · 수정

팔렛트 이동

🔓 해제 모드에서만 이동이 가능합니다. 탭바의 [잠금/해제] 버튼으로 전환하세요.
  1. 1이동할 팔렛트가 있는 슬롯을 탭합니다. (상단에 "이동중" 표시)
  2. 2이동할 빈 슬롯을 탭합니다. → 즉시 이동 완료
  3. 3취소하려면 같은 슬롯을 다시 탭하거나 [취소] 버튼을 누릅니다.

팔렛트 정보 수정

  1. 1수정할 슬롯을 더블탭(더블클릭) 합니다.
  2. 2수정 모달에서 상품명·수량·유통기한을 변경합니다.
  3. 3[저장]으로 반영 또는 [삭제]로 슬롯을 비웁니다.

되돌리기 (Undo)

상단 툴바의 [↩] 버튼으로 직전 작업을 최대 30단계까지 되돌릴 수 있습니다.

8. 📤 출고 — 출고대기 예약

출고 예정 날짜와 목적지를 미리 등록하고 팔렛트를 선택해두는 방식입니다. 실제 출고 전 확인 후 최종 확정합니다.

출고대기 등록 방법

  1. 1작업 탭 상단 [출고대기] 버튼을 누릅니다.
  2. 2출고일(예: 16)과 입고지(예: 인천14)를 입력하고 [상품선택하기]를 누릅니다.
  3. 3그리드에서 출고할 팔렛트 슬롯을 탭합니다. (주황색 [출] 표시)
  4. 4하단 출고대기 패널에 묶음으로 표시됩니다.

출고 확정 / 철회

버튼동작
확정 (개별)해당 팔렛트 1개를 출고 처리 (슬롯 비워짐)
전체출고같은 묶음(날짜-입고지)의 모든 팔렛트 한 번에 출고 처리
철회 (개별)출고대기에서 해당 팔렛트만 취소 (슬롯 유지)
전체철회같은 묶음의 출고대기 전체 취소
💡
출고대기 상태([출] 표시)의 팔렛트는 다른 직원이 실수로 이동하거나 삭제할 수 없도록 잠겨 있습니다.

9. 📤 출고 — 바로출고

예약 없이 즉시 출고할 때 사용합니다. 슬롯에서 팔렛트를 탭하면 바로 제거됩니다.

  1. 1작업 탭 상단 [출고] 버튼을 누릅니다. (빨간색으로 전환)
  2. 2출고할 팔렛트 슬롯을 탭합니다. → 슬롯이 즉시 비워지고 하단 바로출고 대기 목록에 추가됩니다.
  3. 3하단 패널에서 [출고확정]을 눌러 최종 처리합니다.
  4. 4잘못 탭했다면 [취소]를 눌러 해당 팔렛트를 원래 슬롯으로 되돌립니다.
  5. 5완료 후 [일괄출고]로 한 번에 처리하거나 [닫기]로 패널을 닫습니다.
⚠️
바로출고 버튼을 누른 상태에서는 팔렛트가 즉시 슬롯에서 제거됩니다. 실수했을 경우 반드시 [취소] 버튼으로 복구하거나 상단 [↩ 되돌리기]를 사용하세요.

출고 방식 비교

구분출고대기바로출고

팔렛트 상태출고 확정 전까지 슬롯 유지탭 즉시 슬롯에서 제거
취소 가능✅ 철회 버튼✅ 취소 버튼 (확정 전)

11. 📋 기록 탭 — 히스토리 관리

저장 버튼을 누를 때마다 창고 배치 현황의 스냅샷(snapshot)이 기록됩니다. 과거 시점의 배치 상태를 언제든지 조회하거나 복원할 수 있습니다.

저장 방법

  1. 1입출고 작업 완료 후 상단 툴바의 [저장] 버튼을 누릅니다.
  2. 2현재 날짜와 시간이 라벨로 자동 기록됩니다. (예: 2026.03.28 (토))
  3. 3기록 탭에서 해당 날짜 항목을 확인합니다.

기록 항목 기능

버튼기능
배치도해당 시점의 창고 배치 현황을 지도 형태로 조회
복원해당 시점의 상태로 현재 창고를 되돌림 (주의: 현재 데이터 덮어씀)
삭제해당 기록 항목을 영구 삭제
⚠️
복원 기능은 현재 창고 배치를 과거 시점으로 완전히 덮어씁니다. 현재 상태가 필요하다면 복원 전에 먼저 [저장]을 눌러 백업해두세요.
💡
매일 업무 종료 시 저장하는 습관을 권장합니다. 월말 재고 확인 시 원하는 날짜의 기록을 불러올 수 있습니다.

12. 📢 공지 메모

창고 운영 관련 중요 사항을 메모하여 모든 접속 기기에 동시에 표시하는 기능입니다.

사용 방법

  1. 1메인 화면 상단 [📢 공지] 버튼을 누릅니다.
  2. 2내용을 자유롭게 입력합니다. 입력 즉시 자동 저장됩니다.
  3. 3내용이 있으면 메인 화면 상단에 노란 배너로 표시됩니다.
  4. 4내용을 지우려면 [🗑 지우기] 버튼을 누릅니다.
활용 예시
• 밀크런 12파렛트 11시 출고 나머지 2시
• 오늘 냉동창고 정기점검 — 오후 3시 이후 입고 금지
• 쉽먼트 00파렛트 인천항 출발 예정
💡
공지는 Firebase 클라우드에 저장되어 모든 기기에서 동일하게 표시됩니다. 휴대폰으로 작성하면 창고 내 PC에서도 즉시 확인할 수 있습니다.

13. 데이터 내보내기

CSV (엑셀) 다운로드

  1. 1[현황] 탭으로 이동합니다.
  2. 2우측 상단 [엑셀 다운로드] 버튼을 누릅니다.
  3. 3재고현황_창고이름.csv 파일이 자동 저장됩니다.
  4. 4엑셀 또는 구글시트에서 열면 상품명·유통기한·팔렛트수·수량을 확인할 수 있습니다.

데이터 내보내기 (구글시트 연동)

  1. 1창고 화면 상단 [내보내기] 버튼을 누릅니다.
  2. 2조회할 시트(현재상태 또는 히스토리)를 선택합니다.
  3. 3[데이터 복사하기]를 눌러 클립보드에 복사합니다.
  4. 4구글시트 A1 셀을 클릭 후 Ctrl+V로 붙여넣습니다.

15. 자주 묻는 질문 (FAQ)

Q. 저장을 안 했는데 다른 직원 화면에 반영됐어요. 왜인가요?

입출고 작업(슬롯 수정, 출고 확정 등)은 즉시 Firebase에 저장됩니다. [저장] 버튼은 히스토리 스냅샷을 남기기 위한 용도입니다. 중요한 시점마다 저장해두면 나중에 해당 날짜 배치를 조회할 수 있습니다.

Q. 두 직원이 동시에 같은 창고를 수정하면 데이터가 꼬이나요?

창고 단위로 독립적인 경로에 저장되므로 서로 다른 창고를 동시에 수정할 때는 충돌이 없습니다. 같은 창고를 동시에 수정하는 경우 약 4초마다 최신 데이터를 받아 갱신되므로, 동시 작업은 가급적 피하는 것이 좋습니다.

Q. 유통기한 노란색 경고(임박)는 언제 표시되나요?

유통기한이 오늘 기준 90일(3개월) 이내인 팔렛트에 노란색 [!] 경고가 표시됩니다. 이미 지난 경우 빨간색 [!]으로 표시됩니다.

Q. 비밀번호를 잊어버렸어요.

로그인 화면 하단 "비밀번호를 잊으셨나요?"를 탭합니다. 확인을 누르면 비밀번호가 초기화되고 새로 설정할 수 있습니다. 창고 데이터는 삭제되지 않습니다.

Q. 실수로 팔렛트를 출고했어요. 복구할 수 있나요?

상단 [↩ 되돌리기] 버튼을 누르면 직전 상태로 복구됩니다. 최대 30단계까지 되돌릴 수 있습니다. 단, 앱을 새로고침하면 되돌리기 기록이 초기화됩니다.

Q. 공지가 다른 폰에서 안 보여요.

공지는 Firebase에 저장되어 모든 기기에서 동기화됩니다. 안 보인다면 페이지를 새로고침해보세요. Firebase 연결에 문제가 있을 경우 인터넷 연결 상태를 확인하세요.

Q. 창고를 삭제하면 데이터도 삭제되나요?

⚠️ 주의: 창고를 삭제하면 해당 창고의 모든 데이터(셀 배치, 재고, 히스토리)가 영구 삭제됩니다. 복구가 불가능하므로 삭제 전 반드시 엑셀로 내보내기하여 백업하세요.
