📑 사진 + 정보 → PDF · 엑셀

사진 폴더 + 정보 파일(엑셀/CSV) 업로드 → 자동 매칭 → 우측 6필드 직접 입력/수정 → 📊 엑셀 또는 📑 PDF 저장.
이미지 클릭 시 원본 크기로 확대돼 라벨 글씨 확인 가능. 작성 내용은 그룹 클라우드에 자동 동기(파일명 기준) — 다른 PC에서도 같은 폴더 업로드 시 그대로 이어 작업 가능. 사진 자체는 동기되지 않으니 각 PC에서 폴더 재선택.

📁 사진 폴더 지정 ⚫ 동기 OFF 사진 0 · 정보 0 · 작성 0/0
📋 최초 설정 (한 번만)
  1. console.cloud.google.com 가입 + 새 프로젝트 생성
  2. Billing → 카드 등록 (영구 무료 한도여도 필수)
  3. APIs & Services → Library → Cloud Vision API 검색 → Enable
  4. APIs & Services → Credentials → Create credentials → API key
  5. 발급된 키 클릭 → Application restrictions: HTTP referrers → https://makewon.com/*, https://makechango-wms.web.app/*
  6. API restrictions → Cloud Vision API 만 선택 → 저장
  7. 키(AIza...) 복사해 위에 붙여넣고 💾 저장
💳 비용 — 매월 1,000장 무료, 1,001장부터 1,000장당 약 2,000원. 보통 월간 사용량은 무료 한도 안에서 처리됩니다.

📄 정보 파일 업로드

엑셀(.xlsx) 또는 .csv. 첫 줄은 머리글(컬럼명). 아래 어떤 형식이든 자동 인식합니다.

① 품번 · 품명 — 2열 (가장 간단)
② 바코드 · 품번 · 품명 — 3열
③ 바코드 · 품번 · 품명 · 색상 · 사이즈 · 재질 · 제조국 — 전체
💡 ③ 전체 양식 하나로 통일해도 됩니다. 빈 칸이 있어도 값이 들어있는 칸만 자동으로 골라 활용하니, 매번 양식을 바꿀 필요 없이 ③ 한 가지로 쓰면 편합니다.
🎯 정보 파일은 '절대값'입니다. OCR이 사진에서 글자를 뽑을 때 오차가 있어도, 정보 파일의 품번/품명 중 가장 가까운 값으로 바로잡아 채웁니다. (사진 추출값보다 정보 파일이 우선)

머리글 인식 예 — 바코드/barcode, 품번/style_code, 품명/name, 색상/color, 재질/material, 제조국/origin (영문·한글 모두). 사이즈 등 그 외 컬럼은 메모로 함께 보관됩니다.

🧹 데이터 정리 — 무엇을 하나요?

버튼을 누르면 아래 두 가지를 한 번에 처리합니다. 현재 올린 사진은 그대로 유지됩니다.

① 사진 없는 저장 항목 삭제
  • 지금 올린 사진들의 파일명을 기준으로, 사진이 없는 옛 저장 기록(클라우드·브라우저)을 삭제합니다.
  • 예: 433건이 저장돼 있는데 실제 사진이 277장이면 → 사진 없는 156건 정리.
② 품번·품명 보정 (절대값 기준)
  • 각 사진의 품번/품명을 올린 정보 파일(없으면 내장 카탈로그)의 가장 가까운 값으로 교정.
  • OCR 오탈자(O↔0, l↔I 등)를 자동으로 바로잡습니다.
  • 어느 제품인지 애매하면 건드리지 않고 '미매칭'으로 남겨 수동 확인하게 합니다.
⚠️ 실행 전 요약(삭제 N · 보정 M · 미매칭 K)을 먼저 보여주고, 확인을 눌러야 진행됩니다. 클라우드 삭제는 되돌릴 수 없습니다. (사진 파일 자체는 삭제 안 됨)
